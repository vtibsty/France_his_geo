# France History Geography

一个以“历史地理”为视角制作的法国历史交互网页。它不是单纯罗列王朝、战争和年份，而是把法国从高卢、罗马高卢、法兰克、卡佩王朝、绝对王权、革命帝国、殖民帝国到第五共和国的演变，放回地形、河流、边疆、港口和海外网络之中重新观看。

## 项目在做什么

这个项目试图回答一个问题：法国为什么会长成今天这个样子？

页面以时间线为主轴，结合地图、历史图像、地理线索和版图说明，展示法国历史中的几个关键空间变化：

- 巴黎盆地如何成为王权整合的核心。
- 塞纳、卢瓦尔、罗讷等河流如何连接城市、农业腹地与港口。
- 莱茵、阿尔卑斯、比利牛斯等边疆如何塑造法国的安全想象。
- 大西洋和地中海门户如何把法国带入殖民贸易与全球竞争。
- 从王室领地到殖民帝国，再到现代海外法国，版图如何不断重组。

## 新颖之处

**1. 用空间逻辑讲历史**

传统历史时间线常常按年份推进，这个项目则把每个阶段拆成“地理线索、版图变化、海外与殖民、关键地点”四组信息，让历史事件和地理结构互相解释。

**2. 不是静态文章，而是可浏览的历史地图册**

左侧时间线可以切换不同历史阶段，右侧同步展示对应图像、摘要和地理证据。阅读体验更接近翻阅一份数字地图册，而不是读一篇长文。

**3. 本地化图片资源，避免远程断图**

项目中的关键图片已经放入 `public/assets`，并为部分无法稳定加载的远程资料绘制了本地 SVG 替代图，保证网页在本地预览和部署后都能稳定显示。

**4. 视觉上结合档案、地图和历史展陈**

页面采用纸张底色、地图网格、深色图版、历史图像和信息卡片组合，试图营造一种“档案馆 + 历史地图册”的视觉氛围。

## 技术栈

- Vue 3
- Vite
- CSS Grid / Flexbox
- 本地静态图片与 SVG 资产

## 本地运行

```bash
pnpm install
pnpm run dev
```

默认开发服务器会运行在：

```text
http://127.0.0.1:5173/
```

如果 5173 端口被占用，Vite 会自动切换到其他端口。

## 构建

```bash
pnpm run build
```

构建产物会输出到 `dist/`。

## 项目结构

```text
src/
  App.vue        # 页面内容、时间线数据与交互逻辑
  main.js        # Vue 入口
  style.css      # 整体视觉样式

public/assets/
  generated/     # 生成的法国地理与历史主题图片
  history/       # 历史地图、图像和本地 SVG 替代图
```

## 资料来源

项目中的历史图片与地图主要参考 Wikimedia Commons、Wikipedia 相关条目，以及围绕法国历史地理主题整理的公开资料。页面中的外部链接用于标注资料来源和进一步阅读。
